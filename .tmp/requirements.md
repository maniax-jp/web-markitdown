# 要件定義書: Web-MarkItDown

## 1. 概要
ユーザーがアップロードしたあらゆるドキュメントファイルを、サーバーサイドのAPIを使用せず、ブラウザ内（クライアントサイド）のみで完結してMarkdown形式に変換し、ダウンロード可能にするWebサイト。

## 2. 機能要件
- **ファイルアップロード機能**: 
    - ドラッグ&ドロップおよびファイル選択ダイアログによるファイル投入。
    - 対応形式: PDF, DOCX, XLSX, PPTX, HTML 等（markitdownの対応形式に準拠）。
- **クライアントサイド変換**:
    - Pyodide (Wasm) を利用してブラウザ上で Python 環境を構築。
    - `markitdown` ライブラリを使用して変換処理を実行。
- **結果出力**:
    - 変換後のMarkdownテキストを画面上に表示。
    - 変換結果を `.md` ファイルとしてダウンロード可能にする。
- **UX/UI**:
    - Pythonランタイムおよびライブラリのロード中のプログレス表示。
    - 変換中のLoadingインジケータの表示。
    - エラー発生時の適切なエラーメッセージ表示。

## 3. 非機能要件
- **プライバシー**: すべての処理をローカルで行い、ファイルを外部サーバーに送信しない。
- **互換性**: 最新のモダンブラウザ（Chrome, Firefox, Safari, Edge）で動作すること。
- **パフォーマンス**: Pyodideの初期ロード時間を最小限に抑えるためのキャッシュ戦略の検討。

## 4. 制約事項
- 外部APIエンドポイントを使用しない。
- WebAssembly (Wasm) の制限内で動作させる。
